# DSI Workflow Adoption And Notes Migration

## Scope

This repository now treats `docs/dsi/` as the default home for active workstream
documentation and `docs/agent-notes/` as the home for cross-cutting durable notes.

## Migration Reminders

- `docs/reference-projects.md` predates the DSI workflow and should be migrated into
  the newer structure the next time it needs a material update.
- Future agent-generated notes should not be added as ad hoc standalone docs when they
  belong either to an active DSI workstream or to `docs/agent-notes/`.
- When a cross-cutting note exists because of one active workstream, link back to that
  owning DSI folder so the workstream record stays discoverable.
